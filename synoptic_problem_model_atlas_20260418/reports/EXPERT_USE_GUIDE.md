# Expert Use Guide

This atlas is organized so an expert can move from model to evidence and back again.

## Recommended path

1. Open `data/52_system_model_comparison_scorecard.csv` to see the system-model landscape.
2. Use `data/04_gospel_direction_matrix_4x4.csv` to select a source→target direction.
3. Use `data/20_model_pericope_obligation_ledger.csv` to inspect what that direction/model must explain per unit.
4. For Q/Farrer/Griesbach issues, inspect:
   - `data/30_minor_agreements_catalog_triple_tradition.csv`
   - `data/40_double_tradition_order_catalog_markan_masked.csv`
5. For John, inspect:
   - `data/10_pericope_ontology_computational_and_anchor.csv`
   - `data/20_model_pericope_obligation_ledger.csv` filtered to John pairs.
6. Check `data/70_variant_sensitivity_registry.csv` before using a local exact match as high-value evidence.
7. Use `data/81_conclusion_evidence_counterevidence_cards.yaml` for balanced support/counterevidence summaries.

## What counts as objective here?

More objective:
- verse inventory,
- token intersections,
- chain/block memberships,
- gap counts,
- minor-agreement token counts,
- double-tradition order/index fields,
- variant marker presence.

Mixed:
- model obligation wording,
- response models,
- system-model scorecard,
- redactional profiles.

Interpretive:
- current atlas ratings,
- conclusion cards,
- statements that a model is stronger/weaker overall.

## Main scholarly affordance

The atlas does not merely list parallels. It records *explanatory obligations*: if a model is assumed, what material must be omitted, added, displaced, rewritten, or explained as shared tradition?
